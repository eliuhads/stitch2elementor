## 🎯 What is stitch2elementor?

**stitch2elementor** is an open-source **AI agent skill** for tools like [Antigravity](https://github.com/google-deepmind/antigravity), Cline, or any MCP-compatible agent. It orchestrates a fully automated pipeline that takes UI designs generated by **Google Stitch**, extracts their HTML + Tailwind output, and compiles them into **native Elementor Flexbox JSON** — injecting everything directly into WordPress via a hybrid FTP + PHP pipeline that bypasses WAF restrictions.

> **No Figmentor. No UiChemy. No premium plugins. No manual recreation. Zero cost.**

This skill was born from real production migrations. Every protocol, every safety measure, and every fallback documented here was discovered and battle-tested in live WordPress environments.

```
  ┌─────────────────┐      ┌──────────────────┐      ┌─────────────────────┐      ┌─────────────┐
  │  Google Stitch  │ ───► │  HTML + Tailwind  │ ───► │  Elementor JSON V4  │ ───► │  WordPress  │
  │   AI Design     │      │  Agent Download   │      │  compiler_v4.js     │      │  FTP + PHP  │
  └─────────────────┘      └──────────────────┘      └─────────────────────┘      └─────────────┘
```

> ⚠️ **Active Development:** This skill is functional and production-ready for its core pipeline, and is being continuously improved. Protocols, scripts, and new capabilities are added as new edge cases are discovered in real deployments.

## ✨ Key Features

| Feature | Description |
|---|---|
| 🧱 **Native Flexbox Containers** | Converts Tailwind directly into modern Elementor Flexbox schema. No deprecated Section/Column method. |
| 📐 **FULL+BOXED Auto-Architecture** | Section backgrounds at 100vw, content capped at 1200px max-width — exactly like premium human dev. |
| 📱 **Smart Responsive Mapping** | Resolves Tailwind's Mobile-First logic (`flex-col sm:flex-row`) into Elementor's Desktop-First system. |
| 🔤 **Fluid Typography (`clamp()`)** | Injects CSS `clamp()` at widget level for immediate responsiveness without Global Kit dependency. |
| 💉 **Hybrid FTP + PHP Injection** | Bypasses WAF 406/401 errors entirely — uploads via FTP, executes server-side PHP for direct DB writes. |
| 🏗️ **Native Theme Builder** | Header & Footer injected as `elementor_library` templates with auto-discovered menus and global conditions. |
| 🔄 **Auto Cache Flush** | Automatic CSS regeneration, library sync, and permalink flush after every deployment. |
| 🧹 **Post-Gen Sanitization** | Purges Material Symbol text ghosts, fixes internal links, enforces SEO-friendly slugs. |
| 🖼️ **Automatic Image Sideload** | Stitch CDN image URLs (`lh3.googleusercontent.com`) are detected in the compiled JSON and automatically registered into the WordPress Media Library via native `media_sideload_image()` — no local image handling required. |

## ⚡ Dual Trigger Ecosystem

This skill gives your AI agent two distinct operation modes. Just type one word:

### `go!` — The Web Maestro Pipeline

> Full-site generation, from zero to live WordPress.

The agent acts as a **monolithic site architect**. It reads your BrandBook, generates all pages in Google Stitch, compiles the batch through the V4 JSON compiler, and injects a complete, live website — including native Theme Builder Header and Footer — into WordPress in a single automated run.

**Best for:** New site builds, full redesigns, multi-page projects.

---

### `segment!` — The Modular Surgeon

> Atomic updates without touching the rest of the site.

The agent targets a single component (Hero, Pricing Table, CTA, Footer...), compiles a pure JSON array, and performs a **micro-injection**. Bypasses the full pipeline entirely — fast, precise, and safe.

**Best for:** A/B testing, section redesigns, isolated component migrations.

## 🔧 How It Works

### Pipeline Overview (`go!`)

```
Phase 1 — Pre-Flight Check
  └── Verify MCPs (StitchMCP, wp-elementor-mcp, elementor-mcp) are live

Phase 2 — BrandBook & Design System
  └── Create INFO_BrandBook/ folder for the new project
  └── Add BRANDBOOK_GUIDE.md or identity files to INFO_BrandBook/
  └── Generate design_system.json from brand assets
  └── Generate page_manifest.json (mapping slugs to titles)

Phase 3 — Generate in Google Stitch
  └── Enhance prompts via enhance-prompt skill
  └── Create screens via StitchMCP → Download HTML via curl

Phase 4 — Compile + Sanitize
  └── node compiler_v4.js → N page JSONs + header.json + footer.json
  └── node scripts/fix_material_symbols.js → purge icon text ghosts
  └── Audit image URLs → replace CDN refs with WP Media Library paths

Phase 5 — Inject (Hybrid Autonomous)
  └── node scripts/sync_and_inject.js:
      ├── Upload JSONs + PHP injectors via FTP
      ├── Execute create_hf_native.php → Header/Footer as elementor_library
      ├── Execute inject_all_pages.php → batch page injection (sequential)
      │   └── Auto-sideloads Stitch image URLs into WP Media Library
      ├── ⚠️  IDs shift here — capture new IDs from log
      ├── Execute flush_cache.php → Homepage realign + CSS + Library Sync
      └── Auto-delete PHPs from server (security perimeter)

Phase 6 — Verify + SEO
  └── Validate via read_url_content / MCP get_page_structure
  └── Delegate to Agentic-SEO-Skill for meta, Schema, descriptions
  └── Generate final migration report (pages, IDs, slugs, status)
```

## 🧠 Battle-Tested Protocols

These are hard-won lessons from real production deployments. They are codified as non-negotiable rules inside the skill.

### ⚠️ The ID-Shifting Problem

Every time `sync_and_inject.js` runs and injects pages, **WordPress assigns brand new IDs**. Any `wp_id` you recorded before is immediately obsolete. This is not a bug — it's WordPress behavior.

**The consequence:** If you don't re-set the Homepage after injection, WordPress will point your front page to a non-existent or wrong post ID.

### ✅ The "AHORA SI" Protocol (Confirmed Success Flow)

```
Step 1 → Run sync_and_inject.js → IDs change in WordPress
Step 2 → Capture the NEW Homepage ID (from injection log or MCP get_pages)
Step 3 → Update 'home_id' in page_manifest.json
Step 4 → Run flush_cache.php with the new ID
         → Fixes page_on_front + regenerates Elementor CSS + syncs library
```

### 🔧 Config-Only Mode (When NOT to re-inject)

If the site content is stable and you only need to fix the homepage pointer or clear cache, **never run a full re-injection**. Use the dedicated maintenance script instead:

```bash
# Uses home_id from page_manifest.json
node scripts/maintenance_only.js

# Or force a specific ID (example only — actual ID changes after every injection)
node scripts/maintenance_only.js <NEW_HOMEPAGE_ID>
```

This script uploads a self-destructing PHP to the server, sets `page_on_front`, flushes Elementor cache, syncs the library, and auto-deletes the PHP — all without touching a single page's content or ID.

## 🚀 Quick Start

### 1. Clone the skill

```bash
git clone https://github.com/eliuhads/stitch2elementor

# Drop into your agent's skills directory
# For Antigravity:
cp -r stitch2elementor /path/to/.agent/skills/
```

### 2. Install dependencies

```bash
cd stitch2elementor
npm install
```

### 3. Install MCP Servers

```bash
npm i -g wp-elementor-mcp
npm i -g elementor-mcp
```

> ⚠️ Install globally to prevent timeout errors on long pipelines.

### 4. Configure your `.env`

```env
FTP_HOST=ftp.yoursite.com
FTP_USER=your_ftp_user
FTP_PASS=your_ftp_password
FTP_REMOTE_DIR=/public_html/v9_json_payloads/
WP_BASE_URL=https://yoursite.com
INJECT_SECRET=your_secret_token
```

### 5. Configure WordPress credentials

Edit your agent's `mcp_config.json`:

```json
{
  "wp-elementor-mcp": {
    "url": "https://yoursite.com",
    "username": "your_wp_user",
    "app_password": "xxxx xxxx xxxx xxxx xxxx xxxx"
  }
}
```

> Generate Application Passwords at: **WP Admin → Users → Profile → Application Passwords**

### 6. Customize `design_system.json`

Use `design_system_template.json` as your starting point. Configure colors, fonts, typo scale, and logo URL. The compiler loads this automatically at runtime.

### 7. Trigger the pipeline

```
go!       ← Full site build
segment!  ← Single component injection
```

> ⚠️ **ID Shifting Warning:** Every injection via `sync_and_inject.js` assigns **new WordPress IDs** to all pages. Previous IDs become immediately obsolete. After any injection, you **must** capture the new Homepage ID and run `flush_cache.php` to realign the front page. For maintenance without re-injection, use `maintenance_only.js`. See [Battle-Tested Protocols](#-battle-tested-protocols) for the full protocol.

## 📦 Internal Tools — `scripts/`

| Script | Type | Purpose |
|---|---|---|
| `compiler_v4.js` | Node.js | Core DOM walker. Transpiles HTML + Tailwind → Elementor JSON with `clamp()` typography. |
| `sync_and_inject.js` | Node.js | Main orchestrator: FTP upload → PHP execution → cache flush → auto-cleanup. |
| `maintenance_only.js` | Node.js | **Config-Only mode**: realigns Homepage + flushes cache without re-injecting content. |
| `create_hf_native.php` | PHP | Creates Header/Footer as `elementor_library` CPT with global display conditions. |
| `inject_all_pages.php` | PHP | Batch page injector with sequential processing and manifest support. |
| `flush_cache.php` | PHP | Sets `page_on_front`, regenerates Elementor CSS, syncs library, flushes permalinks. |
| `robust_inject_template.php` | PHP | Template injector for Global Kit / BrandBook styles. |
| `fix_material_symbols.js` | Node.js | Purges literal text ghosts from Material Symbols CSS fallbacks. |
| `fix_slugs.js` | Node.js | Normalizes page slugs to match the manifest after injection. |

## 📐 Non-Negotiable Rules

```
🚫  No local browsers (Playwright, Chromium, browser_subagent)
    → Use: read_url_content  /  REST API via MCP  /  curl

🚫  No Elementor JSON wrapper objects
    → Output MUST be: [{...}]  — plain array only, always

🚫  No concurrent REST API requests
    → Injection is strictly sequential: wait for HTTP 200 before next page

⚡  Hybrid Injection is the ONLY deployment path
    → sync_and_inject.js handles FTP + PHP + Cleanup — nothing else

🔄  Always flush cache + realign Homepage after any injection
    → ID-Shifting is inevitable. Run the "AHORA SI" protocol every time.

🔧  Config-Only for maintenance
    → Use maintenance_only.js — never re-inject just to fix the homepage pointer
```

## 🙋 FAQ

**Q: Does this work with Elementor Free or only Pro?**  
A: Core JSON injection works with Elementor Free. Flexbox Containers require Elementor 3.6+. The `nav-menu` widget used in auto-generated headers requires Elementor Pro.

**Q: What is this "ID-Shifting" problem?**  
A: Every time you re-inject pages through the pipeline, WordPress creates new post IDs (it doesn't update existing ones). Your previously noted IDs are immediately stale. Always run `flush_cache.php` with the new Homepage ID after any injection. See [Battle-Tested Protocols](#-battle-tested-protocols).

**Q: I get 406 Not Acceptable or 401 Unauthorized on the WP REST API.**  
A: Expected behavior. ModSecurity/WAF blocks large JSON payloads. The pipeline uses **Hybrid FTP + PHP Injection** by default to bypass the HTTP firewall entirely. No configuration change needed — just make sure FTP credentials in `.env` are correct.

**Q: My homepage is broken after injection. What do I do?**  
A: Run `node scripts/maintenance_only.js [new_homepage_id]`. This is the Config-Only maintenance mode — it sets WordPress's `page_on_front`, regenerates Elementor CSS, syncs the library, and cleans up the injected PHP automatically. No content is modified.

**Q: Can I use this without Google Stitch?**  
A: The `segment!` mode and `compiler_v4.js` work with any valid HTML + Tailwind input. The `go!` mode is tightly coupled to the Stitch → HTML generation flow.

**Q: Why not use Figmentor or UiChemy?**  
A: Both are paid plugins with per-site licensing. This skill is free, open-source, and produces native Elementor JSON with no plugin dependency at runtime. The output is indistinguishable from hand-built Elementor pages.

**Q: Is this production-ready?**  
A: Yes, for its core pipeline. It's also in **active development** — new capabilities are added regularly as real-world edge cases surface. Check `CHANGELOG.md` for the latest.
