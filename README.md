## 🎯 What is stitch2elementor?

**stitch2elementor** is an open-source **AI agent skill** for tools like [Antigravity](https://github.com/google-deepmind/antigravity), Cline, or Claude. It orchestrates a fully automated pipeline that takes UI designs generated by **Google Stitch**, extracts their HTML + Tailwind output, and compiles them into **native Elementor Flexbox JSON** — then injects everything directly into your WordPress site via a hybrid FTP + PHP injection pipeline that bypasses WAF restrictions.

> **No Figmentor. No UiChemy. No premium plugins. No manual recreation. Zero cost.**

```
  ┌─────────────────┐      ┌──────────────────┐      ┌─────────────────────┐      ┌─────────────┐
  │  Google Stitch  │ ───► │  HTML + Tailwind  │ ───► │  Elementor JSON V4  │ ───► │  WordPress  │
  │   AI Design     │      │  Agent Download   │      │  compiler_v4.js     │      │  FTP + PHP  │
  └─────────────────┘      └──────────────────┘      └─────────────────────┘      └─────────────┘
```

## ✨ Key Features

| Feature | Description |
|---|---|
| 🧱 **Native Flexbox Containers** | Converts Tailwind directly into modern Elementor Flexbox schema. No deprecated Section/Column method. |
| 📐 **FULL+BOXED Auto-Architecture** | Section backgrounds at 100vw, content capped at 1200px max-width — exactly like premium human dev. |
| 📱 **Smart Responsive Mapping** | Resolves Tailwind's Mobile-First logic (`flex-col sm:flex-row`) into Elementor's Desktop-First system. |
| ⚡ **Zero Browser Overhead** | All validations via REST API and `read_url_content`. Playwright/Chromium strictly forbidden. |
| 🖼️ **Native Image Preservation** | 100% resolution mapping from Google Stitch to Elementor maintaining pure UI structure. |
| 🧹 **Post-Gen Sanitization** | Purges Material Symbol text ghosts, fixes internal links, enforces SEO-friendly slugs. |
| 💉 **Hybrid FTP + PHP Injection** | Bypasses WAF 406 errors entirely — uploads via FTP, executes server-side PHP for direct DB writes. |
| 🏗️ **Native Theme Builder** | Header & Footer injected as `elementor_library` templates with auto-discovered menus and global conditions. |
| 🔄 **Auto Cache Flush** | Automatic CSS regeneration, library sync, and permalink flush after every deployment. |
| 🔍 **Agentic SEO** | Delegates on-page validation to `Agentic-SEO-Skill` (meta titles, Schema, descriptions). |

## ⚡ Dual Trigger Ecosystem

This skill gives your AI agent two distinct operation modes. Just type one word:

### `go!` — The Web Maestro Pipeline

> Full-site generation from scratch.

The agent acts as a **monolithic site architect**. It reads your BrandBook, optimizes assets, generates all pages in Google Stitch sequentially, compiles the batch through the V4 JSON compiler, and injects an entire live website — including native Theme Builder Header and Footer templates — into WordPress.

**Best for:** New site builds, full redesigns, multi-page projects.

---

### `segment!` — The Modular Surgeon

> Atomic updates without touching the rest of the site.

The agent targets a single component (Hero, Pricing Table, CTA, Footer...), compiles a pure raw JSON array, and performs a **micro-injection**. Bypasses the full pipeline entirely.

**Best for:** A/B testing, incremental improvements, isolated component work.

## 🔧 How It Works

### Pipeline Overview (`go!`)

```
Phase 1 — Environment Check
  └── Verify MCPs (StitchMCP, wp-elementor-mcp, elementor-mcp) are live

Phase 2 — BrandBook + Assets
  └── Read INFO_BrandBook/ → Extract HEX colors + typography
  └── Generate page_manifest.json + design_system.json

Phase 3 — Generate in Google Stitch
  └── Create screens via StitchMCP → Download HTML via curl

Phase 4 — Compile + Inject (Hybrid Autonomous)
  └── Run compiler_v4.js → 20 page JSONs + header.json + footer.json
  └── Execute sync_and_inject.js:
      ├── Upload JSONs + PHP injectors via FTP
      ├── Execute create_hf_native.php (Header/Footer as elementor_library)
      ├── Execute inject_all_pages.php (batch page injection)
      ├── Execute flush_cache.php (CSS + Permalinks + Library Sync)
      └── Auto-delete PHPs from server for security

Phase 5 — Verification
  └── Validate via read_url_content / MCP structure tools
  └── Delegate to Agentic-SEO-Skill for meta, Schema, descriptions
```

## 🚀 Quick Start

### 1. Install the skill

Drop this repo into your agent's skills directory:

```bash
# For Antigravity
cp -r stitch2elementor ~/.antigravity/skills/

# For Cline
cp -r stitch2elementor ~/.cline/skills/
```

### 2. Install MCP Servers

```bash
npm i -g wp-elementor-mcp
npm i -g elementor-mcp
```

> ⚠️ Install globally to prevent timeout errors on long pipelines.

### 3. Configure WordPress Authentication

Edit your agent's `mcp_config.json` with your WordPress URL and an **Application Password**:

```json
{
  "wp-elementor-mcp": {
    "url": "https://yoursite.com",
    "username": "your_wp_user",
    "app_password": "xxxx xxxx xxxx xxxx xxxx xxxx"
  }
}
```

> Generate an Application Password at: **WP Admin → Users → Profile → Application Passwords**

### 4. Configure FTP (for Hybrid Injection)

Add FTP credentials to `sync_and_inject.js` or your `.env`:

```
FTP_HOST=ftp.yoursite.com
FTP_USER=your_ftp_user
FTP_PASS=your_ftp_password
FTP_ROOT=/public_html
```

### 5. Trigger the pipeline

```
go!       ← Full site build
segment!  ← Single component injection
```

## 📦 Internal Tools — `scripts/`

| Script | Type | Purpose |
|---|---|---|
| `compiler_v4.js` | Node.js | Core DOM walker. Transpiles HTML + Tailwind into Elementor JSON. |
| `sync_and_inject.js` | Node.js | Orchestrator: FTP upload → PHP execution → cache flush → cleanup. |
| `create_hf_native.php` | PHP | Creates Header/Footer as `elementor_library` with display conditions. |
| `flush_cache.php` | PHP | Regenerates CSS, syncs library, flushes permalinks. |
| `robust_inject_template.php` | PHP | Template injector for Global Kit/BrandBook styles. |
| `fix_material_symbols.js` | Node.js | Purges literal text ghosts from Material Symbols CSS fallbacks. |
| `fix_slugs.js` | Node.js | Normalizes page slugs to match the manifest. |
| `ftp_injector.js` | Node.js | Alternative standalone FTP utility. |

## 📐 Critical Rules (Non-Negotiable)

```
🚫  No local browsers (Playwright, Chromium, browser_subagent)
    → Use: read_url_content / REST API via MCP

🚫  No Elementor JSON wrapper objects
    → Output MUST be: [{...}] — plain array only

🔄  Always flush cache after injection
    → flush_cache.php is mandatory as final step
```

## 🙋 FAQ

**Q: Does this work with Elementor Free or only Pro?**
A: The core JSON injection works with Elementor Free. Flexbox Containers require Elementor 3.6+. The `nav-menu` widget used in headers requires Elementor Pro.

**Q: What happens if Stitch generates non-semantic HTML?**
A: Use the `enhance-prompt` skill to refine your Stitch directives. The `docs/gotchas.md` covers common output patterns and how to handle them.

**Q: I get a 406 Not Acceptable or 401 Unauthorized via WP REST API.**
A: This is ModSecurity/WAF blocking large JSON payloads. The pipeline uses **Hybrid FTP + PHP Injection** by default, which bypasses the HTTP firewall completely. No configuration change needed.

**Q: Can I use this without Google Stitch?**
A: The `segment!` mode can work with any valid HTML + Tailwind input, not just Stitch output. The `go!` mode is tightly coupled to the Stitch generation flow.

**Q: Why not just use Figmentor or UiChemy?**
A: Both are paid plugins with per-site licensing. This skill is free, open-source, and produces native Elementor JSON — no plugin dependency at runtime.
