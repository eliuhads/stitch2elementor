## 🎯 What is stitch2elementor?

**stitch2elementor** is an open-source **AI agent skill** for tools like [Antigravity](https://github.com/google-deepmind/antigravity), Cline, or Claude. It orchestrates a fully automated pipeline that takes UI designs generated by **Google Stitch**, extracts their HTML + Tailwind output, and compiles them into **native Elementor Flexbox JSON** — then injects everything directly into your WordPress site via REST API.

> **No Figmentor. No UiChemy. No premium plugins. No manual recreation. Zero cost.**

```
  ┌─────────────────┐      ┌──────────────────┐      ┌─────────────────────┐      ┌─────────────┐
  │  Google Stitch  │ ───► │  HTML + Tailwind  │ ───► │  Elementor JSON V4  │ ───► │  WordPress  │
  │   AI Design     │      │  Agent Download   │      │  compiler_v4.js     │      │  REST API   │
  └─────────────────┘      └──────────────────┘      └─────────────────────┘      └─────────────┘
```

## ✨ Key Features

| Feature | Description |
|---|---|
| 🧱 **Native Flexbox Containers** | Converts Tailwind directly into modern Elementor Flexbox schema. No deprecated Section/Column method. |
| 📐 **FULL+BOXED Auto-Architecture** | Section backgrounds at 100vw, content capped at 1200px max-width — exactly like premium human dev. |
| 📱 **Smart Responsive Mapping** | Resolves Tailwind's Mobile-First logic (`flex-col sm:flex-row`) into Elementor's Desktop-First system. |
| ⚡ **Zero Browser Overhead** | All validations via REST API and `curl`. Playwright/Chromium strictly forbidden to save resources. |
| 🖼️ **Native Image Preservation** | 100% resolution mapping from Google Stitch to Elementor maintaining pure UI structure. |
| 🧹 **Post-Gen Sanitization** | Purges Material Symbol text ghosts, fixes internal links, enforces SEO-friendly slugs. |
| 🔍 **Agentic SEO** | Delegates on-page validation to `Agentic-SEO-Skill` (meta titles, Schema, descriptions). |

## ⚡ Dual Trigger Ecosystem

This skill gives your AI agent two distinct operation modes. Just type one word:

### `go!` — The Web Maestro Pipeline

> Full-site generation from scratch.

The agent acts as a **monolithic site architect**. It reads your BrandBook, optimizes assets, generates all pages in Google Stitch sequentially, compiles the batch through the V4 JSON compiler, and injects an entire live website — including Global Headers and Footers — into WordPress.

**Best for:** New site builds, full redesigns, multi-page projects.

---

### `segment!` — The Modular Surgeon

> Atomic updates without touching the rest of the site.

The agent targets a single component (Hero, Pricing Table, CTA, Footer...), compiles a pure raw JSON array, and performs a **micro-injection**. Bypasses the full pipeline entirely.

**Best for:** A/B testing, ModSecurity WAF-safe updates, incremental improvements, isolated component work.

## 🔧 How It Works

### Pipeline Overview (`go!`)

```
Phase 1 — Environment Check
  └── Verify MCPs (StitchMCP, wp-elementor-mcp, elementor-mcp) are live

Phase 2 — BrandBook + Assets
  └── Read INFO_BrandBook/ → Extract HEX colors + typography
  └── Generate page_manifest.json

Phase 3 — Generate in Google Stitch
  └── Create project → Apply Design System → Generate screens (>1280px Desktop)
  └── Pause for user approval if needed

Phase 4 — Compile V4.1
  └── Download HTML via curl (NOT read_url_content — full Tailwind classes required)
  └── Run compiler_v4.js → Extract native Elementor JSON templates
  └── Output: FULL+BOXED containers, header.json (with native nav-menu widget), footer.json
  └── Global elements (<nav>, <footer>) are automatically stripped from standard pages to prevent duplication.

Phase 5 — Inject into Elementor
  └── Create draft pages → Confirm Elementor Canvas layout with user
  └── Inject JSON SEQUENTIALLY via update_page_from_file (wait HTTP 200 per page)

Phase 6 — Post-Processing + SEO
  └── fix_material_symbols.js → fix_internal_links.js → fix_buttons.js
  └── Delegate to Agentic-SEO-Skill for meta, Schema, descriptions
```

## 🚀 Quick Start

### 1. Install the skill

Drop this repo into your agent's skills directory:

```bash
# For Antigravity
cp -r stitch2elementor ~/.antigravity/skills/

# For Cline
cp -r stitch2elementor ~/.cline/skills/
```

### 2. Install MCP Servers

```bash
npm i -g wp-elementor-mcp
npm i -g elementor-mcp
```

> ⚠️ Install globally to prevent timeout errors on long pipelines.

### 3. Configure WordPress Authentication

Edit your agent's `mcp_config.json` with your WordPress URL and an **Application Password**:

```json
{
  "wp-elementor-mcp": {
    "url": "https://yoursite.com",
    "username": "your_wp_user",
    "app_password": "xxxx xxxx xxxx xxxx xxxx xxxx"
  }
}
```

> Generate an Application Password at: **WP Admin → Users → Profile → Application Passwords**
> See `MCP_CONFIGURATION_GUIDE.txt` for the full configuration block.

### 4. Trigger the pipeline

```
go!       ← Full site build
segment!  ← Single component injection
```

## 📦 Internal Tools — `scripts/`

| Script | Purpose |
|---|---|
| `compiler_v4.js` | Core AST-like DOM walker. Transpiles HTML tags + Tailwind classes into Elementor widget schema. |
| `fix_material_symbols.js` | Purges literal text ghosts emitted by CSS icon fallbacks (e.g. `"arrow_forward"`). |
| `fix_internal_links.js` | Updates all internal href values to match final production URLs from the manifest. |

## 📐 Critical Rules (Non-Negotiable)

```
🚫  No local browsers (Playwright, Chromium, browser_subagent)
    → Use: curl / Invoke-WebRequest / read_url_content / REST API via MCP

🚫  No Elementor JSON wrapper objects
    → Output MUST be: [{...}] — plain array only

⚡  Sequential REST API injections only
    → Wait for HTTP 200 per page before proceeding to the next

📖  On layout errors → consult Stitch_Elementor_Guide_GENERAL_V1.md
    → Contains full Tailwind→Elementor responsive mapping dictionary
```

## 🙋 FAQ

**Q: Does this work with Elementor Free or only Pro?**
A: The core JSON injection works with Elementor Free. Flexbox Containers require Elementor 3.6+. Some advanced widget types may need Pro.

**Q: What happens if Stitch generates non-semantic HTML?**
A: Use the `enhance-prompt` skill to refine your Stitch directives. The `Stitch_Elementor_Guide_GENERAL_V1.md` also covers common output patterns and how to handle them.

**Q: Can I use this without Google Stitch?**
A: The `segment!` mode can work with any valid HTML + Tailwind input, not just Stitch output. The `go!` mode is tightly coupled to the Stitch generation flow.

**Q: Why not just use Figmentor or UiChemy?**
A: Both are paid plugins with per-site licensing. This skill is free, open-source, and produces native Elementor JSON — no plugin dependency at runtime.
